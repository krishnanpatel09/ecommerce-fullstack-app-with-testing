<h2>📌 Overview</h2> <p> This project is a full stack e-commerce management system designed to provide a complete online shopping experience. It supports user registration, product browsing, cart management, order processing, and payment simulation, along with admin functionalities for managing products, users, and orders. </p> <p> The system is built using a modern layered architecture with a React frontend and Spring Boot backend, secured using JWT authentication and developed using Test-Driven Development (TDD) and Agile practices. </p>
<h2>🎯 Project Motivation</h2> <p> The project addresses the need for a scalable and secure online retail platform. It focuses on simplifying the shopping experience while providing administrators with efficient tools to manage operations. </p> <ul> <li>Improve user experience in online shopping</li> <li>Ensure secure authentication and authorization</li> <li>Implement maintainable and testable architecture</li> <li>Apply Agile and TDD in a real-world system</li> </ul>
<h2>🚀 Key Features</h2> <ul> <li>User authentication with JWT</li> <li>Role-based access control (Admin, Customer)</li> <li>Product browsing, filtering, and management</li> <li>Shopping cart and checkout system</li> <li>Order tracking and history</li> <li>Admin dashboard for system management</li> <li>Secure REST API integration</li> <li>Comprehensive testing using JUnit and Cucumber</li> </ul>
<h2>🏗️ System Architecture</h2> <p> The application follows a multi-tier architecture using the MVC design pattern. Frontend and backend communicate via REST APIs secured with JWT. </p> <ul> <li><b>Frontend:</b> React (UI, routing, state management)</li> <li><b>Backend:</b> Spring Boot (business logic, APIs)</li> <li><b>Database:</b> SQLite</li> <li><b>Security:</b> Spring Security + JWT</li> </ul> <p> The architecture diagram and workflow are detailed in the project report :contentReference[oaicite:0]{index=0}. </p>
<h2>⚙️ Development Process</h2> <p> The project was developed using <b>Agile methodology</b> with structured sprint planning and task tracking using <b>Jira</b>. </p> <ul> <li>4 iterative sprints with defined goals</li> <li>Daily stand-ups and progress tracking</li> <li>Continuous integration and testing</li> <li>Task management using Jira boards</li> </ul> <p> Agile allowed flexibility, faster feedback integration, and better collaboration across the team :contentReference[oaicite:1]{index=1}. </p>
<h2>🧪 Testing Strategy</h2> <h3>Unit Testing (JUnit)</h3> <ul> <li>Service and controller level testing</li> <li>Validation of business logic and edge cases</li> <li>Cart, product, user, and admin workflows tested</li> </ul> <h3>Behavior-Driven Testing (Cucumber)</h3> <ul> <li>Feature-based testing using real user scenarios</li> <li>Improved readability and validation of user stories</li> </ul> <p> The project follows a strict TDD cycle: write failing tests, implement logic, and refactor :contentReference[oaicite:2]{index=2}. </p>
<h2>📂 Dataset & Data Model</h2> <p> The system uses a structured relational database with the following entities: </p> <ul> <li>Users</li> <li>Products</li> <li>Carts and Cart Items</li> <li>Orders and Order Items</li> </ul> <p> The ER diagram and schema design are included in the report (page 13). </p>
